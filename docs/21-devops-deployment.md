## Hosted Deployment (Render + Vercel)

| Tier      | Platform | URL                                                  |
|-----------|----------|------------------------------------------------------|
| Backend   | Render   | https://stock-analysis-t6yt.onrender.com             |
| Frontend  | Vercel   | https://stock-analysis-orcin-phi.vercel.app          |

### Backend — Render (`render.yaml` blueprint at repo root)

Deploys the FastAPI app from `backend/Dockerfile`, plus managed Postgres and Redis.
Connect the repo in the Render dashboard via **New > Blueprint**.

Environment variables (most are wired automatically by `render.yaml`):

| Var                         | Production value / source                              |
|-----------------------------|--------------------------------------------------------|
| `ENVIRONMENT`               | `production`                                           |
| `DEBUG`                     | `false`                                                |
| `CORS_ORIGINS`              | `https://stock-analysis-orcin-phi.vercel.app`          |
| `HSTS_ENABLED`              | `true` (served over HTTPS)                             |
| `SECRET_KEY`                | generated by Render (`generateValue: true`)            |
| `POSTGRES_*`                | from the managed `stock-analysis-db` database          |
| `REDIS_HOST` / `REDIS_PORT` | from the managed `stock-analysis-redis` instance       |
| `NEPSE_API_URL`             | set in dashboard if using a real NEPSE source          |
| `ML_SERVICE_URL`            | set in dashboard if the ML service is deployed         |

> Setting `CORS_ORIGINS` to a specific origin (not `*`) also flips
> `allow_credentials` to `true` in `backend/main.py`, required for cookie/JWT auth.

### Frontend — Vercel (`frontend/vercel.json`)

`NEXT_PUBLIC_*` variables are **inlined at build time**, so they must be set in the
Vercel project (**Settings > Environment Variables**) before/at build — they are not
read at runtime:

| Var                       | Production value                                       |
|---------------------------|--------------------------------------------------------|
| `NEXT_PUBLIC_API_BASE_URL`| `https://stock-analysis-t6yt.onrender.com/api/v1`      |
| `NEXT_PUBLIC_WS_URL`      | `wss://stock-analysis-t6yt.onrender.com/ws/market`     |
| `NEXT_PUBLIC_ML_API_URL`  | `https://stock-analysis-t6yt.onrender.com`             |

### Local development

Copy the example env files and uncomment the local block:
- `backend/.env.example` → `backend/.env` (uncomment `localhost` / `development` lines)
- `frontend/.env.local.example` → `frontend/.env.local` (uncomment `localhost` lines)
